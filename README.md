# По реке — браузерная мультиплеерная игра

Браузерная реализация настольной игры. 2–6 игроков, пошаговые ходы, маршрут по реке от старта до финиша.

## Требования

- [Node.js](https://nodejs.org/) версии 18 или выше

## Установка

```bash
cd C:\IDE\River
npm install
```

## Запуск

```bash
npm start
```

Сервер запустится на `http://localhost:3000`.

Для разработки (автоперезапуск при изменении файлов):

```bash
npm run dev
```

## Как играть

### Создать игру

1. Открыть `http://localhost:3000` в браузере
2. Ввести имя и нажать **«Создать игру»**
3. Скопировать ссылку из адресной строки (вида `http://localhost:3000/game.html?room=XXXXXXXX`)
4. Отправить ссылку другим игрокам

### Присоединиться

Открыть полученную ссылку в браузере и ввести имя.

### Начало игры

- Каждый игрок тратит монеты на еду, навыки и снаряжение в открывшемся окне
- Нажать **«Готов к игре ✓»** — это применяет покупки и отмечает готовность
- Игра стартует автоматически, когда все игроки готовы (минимум 2)

### Тестирование на одном компьютере

Открыть несколько вкладок браузера с одной и той же ссылкой, вводя разные имена в каждой вкладке.

## Архитектура

### Серверная часть

**`server.js`**

Точка входа. Поднимает HTTP-сервер (Express) и WebSocket (Socket.io), раздаёт статику из `public/`. При старте один раз загружает карту из CSV. Хранит Map всех активных комнат (`roomId → GameRoom`). Маршрутизирует входящие события сокета (`create_room`, `join_room`, `purchase`, `mark_ready`, `action`, `disconnect`) к методам соответствующей комнаты и после каждого изменения рассылает всем участникам полное состояние игры через событие `state`.

**`src/GameRoom.js`**

Класс одной игровой комнаты. Хранит всё изменяемое состояние сессии: список игроков, текущую фазу (`lobby → pre_game → playing → finished`), номер круга, индекс активного игрока, объект текущего хода. Реализует жизненный цикл:

- `addPlayer` / `reconnect` / `disconnect` — вход и выход игроков
- `purchase` / `markReady` — фаза подготовки перед игрой
- `startGame` / `_advanceTurn` — запуск и передача хода
- `action` — делегирует логику в `GameRules.processAction`, затем проверяет, не пора ли перейти к следующему ходу
- `_updateRevealedSquares` — в режиме `closed` открывает участки карты по позиции лидера

**`src/GameRules.js`**

Чистые функции игровой механики без побочных эффектов (кроме броска кубика). Содержит все игровые константы (начальные ресурсы, лимиты, стоимость снаряжения), а также:

- `createPlayer` — создаёт объект игрока с начальными ресурсами
- `applyPurchases` — списывает монеты и начисляет купленные ресурсы и снаряжение
- `initTurn` — инициализирует объект хода для нового игрока
- `simulateMove` — проверяет путь от текущей позиции на наличие порогов
- `processAction` — большой `switch` по типу действия (`roll_red`, `move`, `obstacle_attempt`, `obstacle_portage`, `night_eat_stock`, `end_night` и др.), мутирует игрока и ход, возвращает список событий
- `computeResults` — определяет итоговые места по навыкам и монетам

**`src/boardLayout.js`**

Загружает `board1.csv` и вычисляет экранные координаты для каждого поля. `computeCoordinates` раскладывает поля вдоль маршрута реки внутри клеток сетки 6×6: для каждой клетки находит точки входа и выхода реки и равномерно распределяет поля между ними (в случае разворота — перпендикулярно). Экспортирует константы размеров холста, которые используются и на клиенте.

### Клиентская часть

**`public/js/lobby.js`**

Логика страницы `index.html`. Обрабатывает создание комнаты (выбор режима `open`/`closed`) и вход по инвайт-ссылке. Сохраняет имя игрока и `roomId` в `sessionStorage`, затем перенаправляет на `game.html`.

**`public/js/game.js`**

Главный клиентский координатор на странице `game.html`. Устанавливает соединение с сервером, принимает события `joined` (получает карту и свой `playerId`), `state` (обновляет доску и UI), `events` (пишет лог и запускает анимацию кубиков). Определяет текущую фазу игры и вызывает нужный рендерер: `renderPreGame`, `renderPlayingPhase` (генерирует набор кнопок действий под текущую фазу хода) или `renderFinished`. Отправляет действия игрока на сервер через `socket.emit('action', ...)`.

**`public/js/board.js`**

Класс `BoardRenderer` — рендерер игрового поля на `<canvas>`. При каждом обновлении состояния перерисовывает:

- фоновую сетку клеток
- линию реки по координатам полей
- кружки полей с номерами и иконками (порог, палатка, магазин, рыбалка, грибы); в режиме `closed` — только открытые участки
- фишки игроков, сгруппированные по позиции с угловым разлётом

Обрабатывает события мыши (`mousemove`, `click`) для передачи хода в тултип.

**`public/js/ui.js`**

Набор вспомогательных функций для DOM-манипуляций:

- `updateResourceBars` / `updateEquipment` — обновляют полосы ресурсов и иконки снаряжения
- `updatePlayerList` — список игроков в боковой панели с подсветкой активного и офлайн-меткой
- `setTurnStatus` / `setActionButtons` — строка статуса и кнопки действий
- `animateDie` — CSS-анимация броска кубика
- `addLog` — лог событий (хранит последние 50 записей)
- `showTooltip` — всплывающая подсказка при наведении на поле
- `initPurchaseModal` — модальное окно закупок перед игрой с live-подсчётом стоимости
- `askSteps` — попап выбора количества шагов при движении

### Данные

**`data/board1.csv`**

Карта маршрута: 90 полей, разбитых на 36 клеток сетки 6×6. Каждая строка описывает одно поле: `field_id`, `square_id`, `column_id`, `row_id`, `rapid_difficulty` (0 = обычное, 1–10 = порог), `tent_capacity`, `is_shop`, `is_fish`, `is_mushroom`.
